# 동작 원리 요약

## 전체 흐름

```
페이지 로드 → 3개 모델 .bin 동시 fetch → 파티클로 렌더링
                                            ↓
                                    스크롤/마우스에 반응
```

---

## 1. 파티클 렌더링

GLB 3D 모델의 버텍스(꼭짓점)를 **점(dot)** 으로 그린다.

```
빌드 타임:  GLB 모델 → 버텍스 좌표 추출 → .bin 파일로 저장 (최대 15,000개)
런타임:     .bin 로드 → 8유닛 정규화 → THREE.Points로 점 렌더링
```

각 점은 원형 그라데이션 텍스처가 적용된 흰색 파티클.
깊이(Z축)에 따라 가까운 파티클은 크게, 먼 파티클은 작게 표시 (커스텀 셰이더).

---

## 2. 패럴랙스 회전

**마우스 스크린 위치**에 따라 오브젝트 전체가 미세하게 기울어진다.

```
마우스 왼쪽 → 오브젝트 왼쪽으로 회전
마우스 오른쪽 → 오브젝트 오른쪽으로 회전
마우스 위 → 오브젝트 위로 기울임
마우스 아래 → 오브젝트 아래로 기울임
```

- 최대 약 17° (0.3 라디안)
- 스크롤이 아닌 **마우스 커서 위치** 기반
- 부드럽게 보간되어 자연스럽게 따라감

---

## 3. 마우스 dome 인터랙션

마우스 포인터가 오브젝트 근처에 오면 **dome(돔) 영역** 안의 파티클이 반응한다.

### 반발 (scatter)

```
        ●  ← 파티클
        ↑ 밀려남 (마우스 반대 방향)
        🖱️ 마우스
```

- 마우스 **반대 방향**으로 밀어냄 (랜덤 아님)
- 돔 중심(마우스 위치)에 가까울수록 강하게, 가장자리는 약하게 (cosine dome 감쇠)
- 마우스가 **움직일 때** 강하게 반응, **멈추면** 서서히 원위치로 복귀

### 공전 (orbit)

- 돔 안의 파티클이 **접선 방향**으로 미세하게 진동
- 마우스 움직임에 비례하여 활성화, 정지 시 감쇠

### 크기 효과 (size effect, 기본 off)

- 돔 중심에 가까운 파티클이 약간 커짐 (볼록 렌즈 느낌)

### 복귀 방식

- **lerp** (기본): 선형 보간으로 부드럽게 원위치
- **spring** (옵션): 스프링 물리로 탄성 바운스 후 원위치

---

## 4. 스크롤 전환 (scatter ↔ reform)

스크롤로 모델 간 전환. **3개 모델이 페이지 로드 시 모두 로드**되어 있고,
스크롤 위치에 따라 각 모델의 파티클이 흩어지거나 모인다.

### 핵심 공식

```
최종 위치 = 원래 위치 + (미리 계산된 랜덤 오프셋 × scatterAmount)
```

- `scatterAmount = 0` → 원래 형태 (모델 보임)
- `scatterAmount = 1` → 완전히 흩어짐 (모델 안 보임)

### 각 파티클의 랜덤 오프셋

모델 로드 시 **한 번만** 계산하여 저장. 매 프레임 새로 계산하지 않음.

```
방향: 구형 균일 랜덤 (모든 방향 동일 확률)
거리: 5 ~ 15 유닛
```

→ 같은 파티클은 항상 **같은 경로**로 흩어졌다 모인다.

### 스크롤 구간별 동작

```
스크롤 0%                                        100%
├── 모델0 (30%) ──┤
             ├── 모델1 (30%) ──┤
                          ├── 모델2 (30%) ──┤
```

각 모델의 30% 구간 내부:

```
│  진입 20%  │     고정 60%      │  퇴장 20%  │
│            │                   │            │
│ scatter 1→0│  scatter = 0      │ scatter 0→1│
│ opacity 0→1│  opacity = 1      │ opacity 1→0│
│ 모임+페이드인│  형태 유지        │ 흩어짐+페이드아웃│
```

### 전환 예시 (모델0 → 모델1)

```
시간 →

모델0:  ████████████████▓▓▓▓░░░░
        [고정]          [퇴장]
        scatter=0       scatter 0→1, opacity 1→0

모델1:              ░░░░▓▓▓▓████████████████
                    [진입]          [고정]
                    scatter 1→0     scatter=0
                    opacity 0→1
```

- 모델0의 파티클이 흩어지면서 사라짐
- 동시에 모델1의 파티클이 모여들면서 나타남
- 겹치는 구간에서 크로스페이드 효과

### 첫 모델 예외

스크롤 0%(페이지 최상단)에서는 scatter 없이 **즉시 형태가 표시**된다.
사용자가 처음 페이지를 열었을 때 바로 모델이 보이도록.

## 데이터 흐름 요약

```
매 프레임 update() 호출
  │
  ├─ scrollProgress (0~1) 계산
  │   └─ calculatePhase() → { opacity, scatterAmount }
  │
  ├─ 파티클 위치 계산 (15,000개 루프)
  │   │
  │   ├─ 기본 위치 = originalPos + scatterOffset × scatterAmount
  │   │
  │   ├─ 마우스 dome 체크
  │   │   ├─ dome 안 → repel + orbit 계산 → mouseOffset 갱신
  │   │   └─ dome 밖 → mouseOffset을 0으로 감쇠
  │   │
  │   └─ 최종 위치 = 기본 위치 + mouseOffset
  │
  ├─ 패럴랙스 회전 적용 (오브젝트 전체)
  │
  └─ opacity 적용 → 렌더링
```
